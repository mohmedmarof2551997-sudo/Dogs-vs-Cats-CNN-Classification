## 🎯 Overview

A binary image classifier (Cat vs Dog) built as a Convolutional Neural Network trained **entirely from scratch** — no pretrained/transfer-learning weights. The pipeline covers real-image EDA, a 4-block CNN, augmentation, callback-driven training, rigorous leak-free evaluation, Grad-CAM explainability, and a deployed Streamlit app.

**Target:** binary classification — `Cat` (0) or `Dog` (1)
**Source:** [Kaggle — Dog and Cat Classification Dataset](https://www.kaggle.com/datasets/bhavikjikadara/dog-and-cat-classification-dataset)

## 📦 Dataset

- **25,000 images** total — 12,499 cats + 12,499 dogs
- Split **80% train / 10% validation / 10% test**:

| Split | Cats | Dogs |
|---|---|---|
| Train | 9,999 | 9,999 |
| Validation | 1,249 | 1,249 |
| Test | 1,251 | 1,251 |

- Images resized to 224×224×3

## ✅ Verifying a Leak-Free Split

Because a random train/val/test split can silently overlap if the split logic is ever re-run without clearing prior output, every split in this project is generated by wiping the target directory first, splitting exactly once, and then asserting the totals add back up to the original source count:

```python
assert total_cats == len(cat_files), "Cat split doesn't add up — still overlapping!"
assert total_dogs == len(dog_files), "Dog split doesn't add up — still overlapping!"
```

No split is trusted until its totals are verified — this is what makes the test accuracy below a genuinely held-out number, not an inflated one.

## 🔍 Exploratory Data Analysis

Before any modeling, the notebook inspects:
- Class balance across train/validation/test (bar chart)
- Random sample grids of raw cat and dog photos
- Raw image shape and the individual R / G / B channel decomposition of a sample image
- A batch of augmented training images (rotation, shift, zoom, shear, horizontal flip) to sanity-check what the network actually sees during training

## 🧠 Model Architecture

```
Input (224 × 224 × 3)
 → Conv2D(64)  → BatchNorm → MaxPooling2D
 → Conv2D(128) → BatchNorm → MaxPooling2D
 → Conv2D(256) → BatchNorm → MaxPooling2D
 → Conv2D(256) → BatchNorm → MaxPooling2D
 → Flatten
 → Dense(128) → BatchNorm → Dropout(0.3)
 → Dense(1, sigmoid)
```

~7.4M trainable parameters. The filter counts, dense layer size, dropout rate, and learning rate reflect a prior Keras Tuner Hyperband search across candidate values for each.

## ⚙️ Training Setup

| Setting | Value |
|---|---|
| Optimizer | Adam, learning rate 0.0005 |
| Loss | Binary crossentropy |
| Batch size | 32 |
| Max epochs | 30 |
| EarlyStopping | monitor=`val_accuracy`, patience=8, restore best weights |
| ReduceLROnPlateau | monitor=`val_loss`, factor=0.5, patience=3, min_lr=1e-9 |
| Augmentation | rotation, width/height shift, zoom, shear, horizontal flip |

## 🏆 Results

**Final test accuracy: 96.80%** (loss 0.0917), evaluated on the fully held-out 2,502-image test set.

| Class | Precision | Recall | F1-Score | Support |
|---|---|---|---|---|
| Cat | 0.98 | 0.96 | 0.97 | 1,251 |
| Dog | 0.96 | 0.98 | 0.97 | 1,251 |
| **Accuracy** | | | **0.97** | 2,502 |

Misclassified: **80 / 2,502 (3.2%)**

> ⚠️ **On "100% accuracy":** a CNN scoring a genuine 100% on a real, held-out photo test set is almost always a sign of data leakage, not a win. 96.80% on a verified, leak-free split is the honest, reproducible number for this setup.

## 🔎 Misclassified Examples

Rather than stopping at the accuracy number, the notebook pulls and visually inspects the actual misclassified test images — the fastest way to spot patterns in *why* a model fails (unusual angles, multiple animals in frame, poor lighting, breed ambiguity) instead of just measuring that it does.

## 🔍 Explainability — Grad-CAM

Every prediction can additionally be inspected with **Grad-CAM** (Gradient-weighted Class Activation Mapping, deployed in the companion Streamlit app), which highlights exactly which pixels influenced the model's decision — turning a black-box prediction into a visual, checkable one.
